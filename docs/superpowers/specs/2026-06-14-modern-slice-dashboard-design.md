# 现代切片工作台设计

## 目标

以 `C:\Users\zk\Downloads\bilive_modern_dashboard_preview.png` 和
`C:\Users\zk\Downloads\bilive_modern_dashboard.html` 为视觉基准，重构现有切片页面，
让录播选择、视频预览、弹幕峰值定位和人工判断形成一个清晰的桌面工作台。

## 范围

- 只修改 `frontend/index.html`、`frontend/styles.css` 和必要的
  `frontend/app.js` 展示逻辑。
- 保留当前 Dashboard API、Windows Worker API 触发方式和 Pi/Windows 运行边界。
- 保留现有任务诊断、任务队列、录播列表、片段操作和键盘快捷键。
- 保留真实弹幕密度图及片段覆盖层；点击覆盖层仍应跳转视频并选中片段。
- 不引入 React、Vue、构建工具或外部字体依赖。

## 页面结构

页面采用参考稿的桌面信息架构：

1. 76px 深色图标侧栏。
2. 76px 顶栏，展示页面标题、Worker 状态、刷新和启动切片操作。
3. 顶部概览卡，合并当前文件、处理进度、诊断阶段和任务统计。
4. 主工作区为三栏：
   - 左栏：录播审核队列、房间筛选和状态筛选。
   - 中栏：真实视频播放器、弹幕密度图、当前片段指标。
   - 右栏：AI 判断摘要、标题、描述、原因、裁剪区间和操作按钮。
5. 详细任务队列保留在工作区下方的可折叠区域。

## 数据与交互

- 现有元素 ID 保持稳定，避免破坏 `frontend/app.js` 的数据绑定。
- 顶部概览复用 `/api/slice-progress` 和 `/api/slice-diagnostics`。
- 左侧列表继续复用 `/api/source-recordings`，选择录播后请求详情。
- `density-chart` 继续由 `renderDensityChart()` 绘制真实密度和片段覆盖层。
- 人工保留、丢弃、保存区间、重新判断和渲染继续调用现有片段 API。
- 页面在窄桌面下允许主区域纵向滚动；三栏在小屏断点下改为单栏。

## 视觉原则

- 背景使用浅蓝灰，卡片使用白色、16px 圆角和轻阴影。
- 主色使用参考稿蓝色，成功、警告、失败状态使用绿、琥珀和红色。
- 信息密度接近参考稿，但不复制其中的模拟播放器、模拟波形或装饰占位图。
- 图标继续使用项目已有的图标渲染方式，不新增网络资源。

## 验收标准

- 主要布局在 1920x1080 下与参考稿的比例和视觉层级一致。
- 所有现有关键 DOM ID、API 字符串和操作按钮仍存在。
- 弹幕密度图可见，并保留真实曲线、保留片段和失败片段覆盖层。
- Python 前端契约测试、完整 pytest、compileall 和 `pip check` 通过。
- 浏览器中无明显横向溢出、遮挡、不可操作控件或控制台错误。
